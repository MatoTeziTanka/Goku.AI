- Identify purpose, development status, file organization
- Find misplaced, orphaned, EOL files
- Identify consolidation opportunities
- Log all findings ‚Üí `{REPO}-CODE-AGENT-ANALYSIS.md`

**Step 2: Review Agent**
- Review Code Agent's analysis
- Validate findings
- Identify gaps or disagreements
- Add additional insights
- Log review ‚Üí `{REPO}-REVIEW-AGENT-VALIDATION.md`

**Step 3: Azure GPT-4.1 API**
- Review both agent analyses
- Build consensus
- Resolve conflicts
- Final recommendations
- Log consensus ‚Üí `{REPO}-AZURE-CONSENSUS.md`

- Create `{REPO}-COMPILED-CHANGES.md` with:
  - All agreed-upon changes
  - All disagreements (with reasoning)
  - Priority levels
  - Implementation plan
